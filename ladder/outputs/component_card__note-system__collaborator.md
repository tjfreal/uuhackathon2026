# Note System
*Foundation layer · built*

It provides the local corpus, CLI, and review habits that the proposal-layer tools depend on.

**What it is**: A local-first markdown-based knowledge system for capturing, reviewing, and resurfacing what you know.

**The problem it solves**: Notes are easy to collect and easy to lose. A plain archive is not the same thing as a usable body of knowledge.

**A concrete example**: the user keeps daily notes in plain text, then uses review passes and local tooling to revisit ideas that would otherwise sink back into the archive.

**How it fits**:
- Surface depends on this corpus to resurface older notes.
- Bridge depends on the same corpus to find cross-domain connections.
- Ladder explains the system so other people can actually enter it.

**Inputs**:
- local markdown notes
- daily notes
- inbox captures
- periodic review passes

**Outputs**:
- a living corpus of linked notes
- searchable and reviewable personal knowledge
- raw material for higher-level tools

**Dependencies**:
- local files
- notes CLI

**Current limitations**:
- Review still depends on consistent human habits
- The base system is personal and opinionated rather than universally packaged

**Next rung for you**: Inspect the notes workflow and decide which assumptions are hard requirements versus personal habits.
