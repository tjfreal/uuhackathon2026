Title|||Throughline|||Multi-span semantic search for personal knowledge|||Throughline is the strongest project in the current hackathon set because it solves a real tooling problem and explains itself well in a demo. The short version is that semantic search gets better when you stop pretending one chunk size is always correct.
Statement|||The Core Claim|||Retrieval quality depends on granularity.|||A phrase-level match, a paragraph-level match, and a document-level match are different kinds of relevance. Most semantic search systems hide that distinction. Throughline makes it visible.
Title & Bullets|||What The System Does|||Index the same corpus at multiple span sizes\nQuery all span indices in parallel\nRender results in side-by-side columns\nHighlight repeated hits as throughlines|||The system indexes one corpus several ways, then queries all of them at once. The interface shows where different granularities agree and where they diverge. Agreement across spans becomes a signal instead of an accident.
Title & Bullets|||What Already Exists|||Embedding pipeline in scripts/parse_embeddings.py\nQuery runner in scripts/query_embeddings.py\nDocument-level embeddings in .index/embeddings/\nWorking local Python environment and model stack|||This is not a greenfield project. The core embedding and querying path already exists. That is what makes Throughline a good hackathon candidate: most of the work is extension, integration, and presentation rather than invention from nothing.
Title & Bullets|||What We Need To Build|||Fixed-span indexing support\nMulti-index query execution\nThroughline grouping logic\nA local UI for comparison and demo|||The build is cleanly staged. First parameterize chunking, then query multiple indices, then surface the agreements, then wrap it in a UI. That sequence makes it practical under hackathon constraints.
Section|||Why It Matters|||This is a search tool designed for LLM readers, not just human readers.|||The broader point is not only that the search gets better. It is that knowledge systems should expose what level of context is actually relevant. Throughline turns chunk granularity into something explicit and inspectable.
